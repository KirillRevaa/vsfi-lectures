init SysV

  • 0 - halt (выключает систему)

  • 1 - Single user mode (работа из под одного пользователя)

  • 2 - Multiuser, without NFS (Тоже самое, что и 3, только без сети)

  • 3 - Full multiuser mode

  • 4 - unused

  • 5 - X11 (графический режим)

  • 6 - reboot

inittab

# Уровень по умолчанию
id:3:initdefault:
# Инициализация системы (запускается при загрузке системы)
si:S:sysinit:/etc/rc.d/rc.S
# Скрипт, который запускается в режиме с одним пользователем
#    (уровень 1)
su:1S:wait:/etc/rc.d/rc.K
# Скрипт, который запускается в многопользовательском режиме
rc:23456:wait:/etc/rc.d/rc.M
# Что делать при нажатии комбинации Ctrl-Alt-Del
ca::ctrlaltdel:/sbin/shutdown -t5 -rfn now
l0:0:wait:/etc/rc 0
l1:1:wait:/etc/rc 1
l2:2:wait:/etc/rc 2
l3:3:wait:/etc/rc 3
l4:4:wait:/etc/rc 4
l5:5:wait:/etc/rc 5
l6:6:wait:/etc/rc 6
# Что делать при перебое в питании (выгрузка системы в режиме с
# одним пользователем)
pf::powerfail:/sbin/shutdown -f +5 "THE POWER IS FAILING"
# Если питание успело восстановиться, прекратить выгрузку системы
pg:0123456:powerokwait:/sbin/shutdown -c "THE POWER IS BACK"
# Если питание восстановилось в режиме с одним пользователем,
# то вернуться к многопользовательскому режиму
ps:S:powerokwait:/sbin/init 5
1:23:respawn:/sbin/mingetty tty1
2:23:respawn:/sbin/mingetty tty2
3:23:respawn:/sbin/mingetty tty3
4:23:respawn:/sbin/mingetty tty4

Команды sysV

  • init - управление инциализацией.

  • teleinit <runlevel> - управление процессом init, использовалась в старых дистрибутивах.

  • wall - вывод сообщения пользователя системы

  • halt - выключение компьютера

  • reboot - перезагрузка

  • shutdown - запланированное выключение

  • runlevel - посмотреть текущий уровень

SystemD. JournalD

  • journalctl -f - все события в реальном времени

  • journalctl _UID=<UID> - все события для пользователя

  • journalctl <service> -f - события определнного сервиса в реальном времени

SystemD преимущества

  • Более быстрая загрузка из-за использования параллельных потоков

  • Более надежная загрузка. Если один сервис завис, другие его не дожидаются

  • Более надежная система последовательности, не на основе PID.

  • Удобная система журналирования
